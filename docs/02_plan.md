주의: 선형 모델(E000/E001)에서 확인한 threshold 튜닝 +0.011은 **약한 모델에만 유효**. sqrt 가중으로 학습된 Qwen에는 이득 0이 실측돼 있으므로 기대하지 말 것.

## 딥리서치 2회차 결과 (07-13 오후, 23/25 검증 통과)

기각 목록 밖에서 살아남은 카드 (기대값순):
1. **레이어 프루닝으로 추론 병목 해소 → 블렌드 회수** (최유력, medium): Qwen3-0.6B 마지막 20~30% 레이어 잘라내고 짧은 재학습 → ~6:40대 단일 모델. 그러면 [프루닝 Qwen3 6:40 + 인코더 2:40 ≈ 9:20]이 10분 안에 들어와 **"시간 초과"로 기각됐던 블렌드 +0.0034(OOF)를 되살릴 유일한 경로.** oracle 0.8028 다양성을 예산 안에서 실현. 근거: L3Prune(NAACL WS'25, "30% 프루닝 무손실 + 1000스텝 healing") — 단 3.7~7.5B 임베딩 세팅이라 0.6B 분류 전이는 미검증, fold0 스모크 + bench_infer 실측 필수.
2. **mmBERT-base 이종 앙상블 파트너** (high): ModernBERT 아키텍처(22층, 307M, MIT), **Gemma 2 토크나이저(256k)라 학습실패 이력의 XLM-R 250k unigram SPM과 다른 계보.** XLM-R base 대비 분류 +2~3점(단 저자 자체보고, 한국어 벤치 부재). 단독 대체는 어려움(외삽 0.75~0.76 < Qwen3 0.7679)이나 빠른 추론(인코더급)으로 블렌드 파트너 가치. **⚠️ 256k 대형 vocab(임베딩 197M)이 과거 실패의 진짜 원인이었으면 재발 → 512샘플 암기 스모크 필수.**
3. **balanced softmax = in-training logit-adjusted CE(τ=1)** (부수 실험): 기각된 post-hoc과 별개 기법(학습 목적함수 자체 변경). 구현 ~0비용(로짓에 +log π_y 한 줄). 단 문헌 이득이 극단 tail(≤35샘플)에 집중, 우리 최희소 1273샘플이라 기대 +0~0.004. fold0 게이트 1회.

비권장/기각 (검증 결과):
- **cleanlab 라벨 정리**: 벤더 문서 자체가 "노이즈 test에선 비보장" 경고, 플래그 정밀도 평균 51%, class-agnostic 필터가 희소클래스 오폐기(macro-F1 치명타). **test 라벨이 train과 같은 합성 과정 공유(CV↔LB 갭 0이 방증) → 정정 학습이 LB에서 감점 가능.** CPU 사이드 보수 변형(제거-only)만 여유 시.
- **풀링 교체(last-token vs mean)**: 유일 실측에서 데이터셋마다 승자 뒤집힘(≤0.014), 부호 비일관 → 단독 슬롯 배정 불가.
- 검증 통과 0건(외부 근거 없음, 자체 실험으로만 판단): TAPT/DAPT, EMA/SWA/multi-sample dropout, paraphrase 증강, 직렬화 5개 하위항목, HyperCLOVAX-SEED(라이선스 미확인).
- **미해결 최유력 지점**: oracle 0.8028을 만드는 두 모델의 오류 상보성이 어느 입력 특징(클래스/필드/세션 위상)에서 오는가 — 상위 12팀의 +0.02가 여기 있을 확률이 가장 높은 자체 오류 분석 과제.

## 딥리서치 2회차 교차분석 (GPT 3번째 리포트, 07-13)

두 독립 리서치가 **합의**한 것(신뢰도 상승, 재확인 불필요): TAPT/DAPT 회피, paraphrase 회피, cleanlab 자동정리 반대(수동만), balanced softmax는 1회 A/B 부수, Seesaw 금지·LDAM 후순위.

두 리서치가 **충돌**한 것 + 판정:
- **백본**: 내 리서치=mmBERT, GPT=Granite-97M 우선(XLM-R vocab 재사용 안 함·Apache-2.0·인코더 속도 명시). → 우리는 이미 mmBERT 실측 학습 성공(스모크 통과)·5-fold 진행 중이라 **mmBERT 유지**. Granite-97M은 mmBERT 다양성이 실패할 때의 백업 후보로만 등록.
- **오늘 밤 슬롯**: GPT는 현 Qwen에 "직렬화v2+pooling+EMA/MSD 스택"을 제안. → mmBERT 5-fold 진행 중이라 GPU 재배치 안 함. 단 아래 항목을 **내일 밤 슬롯 후보로 승격**.

GPT 리포트가 내 리서치의 맹점을 찌른 것 (실측이 뒷받침):
- **result_summary 상태 구조화** (신규 승격, 기대 +0.003~0.008): `[RS=OK/EMPTY/ERR]` + `[COUNT=0/1/2_5/6P]` + `[TRUNC]` 같은 **행동 상태 태그**. 기각된 "표면 단서 태그"와 다른 것 — 그건 느슨한 surface hint였고 이건 도구 실행 성패·반환 개수를 직접 표시. **우리 오라클 실측이 지지**: XLM-R이 Qwen을 이기는 유일 클래스가 list_directory(=결과 개수가 결정적). 태그형+짧은 자연어 하이브리드, 수치는 coarse bucket(0/1/2-5/6+). 내일 밤 fold0 게이트(+0.002).
- **EMA/MSD 학습 안정화** (추론 비용 0): 마지막 30~40% step EMA + 분류 헤드 multi-sample dropout. 내일 밤 재학습에 얹기. 기대 +0.001~0.004.
- pooling A/B(last-token vs mean)는 부호 비일관(양 리서치 동의)이라 단독 슬롯 안 배정, 재학습 시 무임승차만.
