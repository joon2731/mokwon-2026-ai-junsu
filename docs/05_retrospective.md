## 3. 통한 것 (왜 통했나)

1. **full-data 재학습** (+0.0035) — 검증용 fold 모델을 그대로 제출하고 있었음. 데이터 20% 추가가 가장 큰 단일 이득.
2. **앙상블 지식 증류** (+0.0020) — 블렌드(OOF 교사 0.7754)를 단일 0.6B에 압축. **서버 10분 추론 벽을 우회하는 유일한 앙상블 활용법.** 실측: 블렌드 OOF 0.7734 ≈ 증류 학생 LB 0.77326 → **증류가 앙상블 이득을 거의 완전히 흡수**.
3. **교사 다양화** (+0.0006) — 교사에 mmBERT(다른 계보 인코더)를 추가하니 학생도 좋아짐. 다양성은 교사 쪽에서 확보하는 게 효율적.
4. **au_bias 강도 재튜닝** (+0.0005, 비용 0) — 이전 세대(XLM-R) 기준으로 튜닝된 prior 보정 강도가 증류 학생에겐 **과교정**이었음. ×0.7이 최적(단봉 확인). **모델이 바뀌면 후처리도 재튜닝해야 한다.**

## 5. 왜 못 넘었나 — R107 오차 진단 (핵심)

## 8. 프로세스 교훈

**잘한 것**
1. **fold0 게이트 규율** — 새 카드는 3.3h fold0 프로브 + 게이트(≥0.7699)로 판정. 6개 카드를 값싸게 걸러냄
2. **CV↔LB 갭 0 확인** — Qwen 계열은 CV가 LB를 그대로 예측(0.7571→0.7576). 이걸 초기에 확인해서 CV 기반 의사결정이 가능했음
3. **모든 기각을 기록** — 같은 실패를 반복하지 않았고, 이 문서의 §4가 그대로 자산이 됨

**아쉬운 것 (가장 중요)**
4. **진단이 너무 늦었다** — R107(오차 상보성 분석)은 **CPU 10분**짜리인데 마감 전날에야 했다. 이걸 첫날 했으면:
   - "nav 초기 스텝 = 정보 한계"를 알고
   - 학습 레버 4개(lr·groupCE·R-Drop·BSM ≈ **GPU 16시간**)를 태우지 않았을 것
   - 그 시간을 §6의 질문(다른 접근)에 썼을 것
   - **교훈: 최적화 전에 진단하라. 병목이 어디인지 모르면 아무리 튜닝해도 헛돈다.**
5. **후처리 재튜닝을 늦게 발견** — au_bias 강도(×0.7)는 재학습 0으로 +0.0005였다. 모델을 바꿀 때마다 후처리 파라미터도 재튜닝 대상이라는 걸 마지막 날에야 깨달음
6. **GPU 하드웨어 한계 무시** — 14시간 무휴식 풀로드 → **PCIe AER 에러(WHEA 17) → GPU lost**. 재부팅으로 복구했지만 실험 체인에 냉각 간격이 필요했다

**의사결정 회고**
- 총 GPU 사용 ≈ 40h. 이 중 **점수를 만든 건 ~10h**(full-data 4.5h + 증류 4.5h×2), **~30h는 기각된 카드**(1.7B 6.5h · Embedding 3.3h · lr 3.3h · groupCE 5.5h · R-Drop 7h · BSM 3.3h · seed777 1.3h)
- 마지막 +0.0005는 **CPU 12분**(au 스윕 패키징)으로 얻었다
- **"큰 GPU 실험"보다 "값싼 진단·후처리 재튜닝"의 시간당 수익률이 압도적으로 높았다**
